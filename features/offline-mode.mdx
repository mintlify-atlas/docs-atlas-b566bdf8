title: Offline mode
description: Automatic audio caching for seamless offline playback

Rotations automatically caches every track you play, enabling offline listening without manual downloads. Once a track streams for the first time, it's permanently available offline.

## How it works

The offline system uses two layers of caching: a service worker for network requests and IndexedDB for audio file storage.

<Info>
Offline mode is completely automatic. There's no "download" button - everything happens in the background as you listen.
</Info>

### Cache layers

<Tabs>
  <Tab title="IndexedDB cache">
    Audio files are stored as Blobs in IndexedDB using the `MusicCache` class. This provides fast access and efficient storage for large audio files.

    ```typescript
    // From site/src/lib/musicCache.ts:3-5
    const DB_NAME = "fucking-music-cache"
    const STORE_NAME = "audio-files"
    const DB_VERSION = 1
    ```
  </Tab>
  
  <Tab title="Service worker cache">
    The service worker intercepts network requests and serves cached responses when offline.

    ```javascript
    // From site/public/sw.js:1-23
    const CACHE_NAME = "fucking-music-v1"
    const PRECACHE_ASSETS = ["/", "/less", "/more"]

    self.addEventListener("fetch", (event) => {
        const url = new URL(event.request.url)
        
        if (url.pathname.startsWith("/api/audio/proxy")) {
            event.respondWith(
                caches.match(event.request).then((cached) => {
                    if (cached) return cached
                    return fetch(event.request).then((response) => {
                        if (response.ok) {
                            const clone = response.clone()
                            caches.open(CACHE_NAME).then((cache) => {
                                cache.put(event.request, clone)
                            })
                        }
                        return response
                    })
                })
            )
        }
    })
    ```
  </Tab>
</Tabs>

## Audio caching flow

When you play a track, Rotations follows this caching strategy:

<Steps>
  <Step title="Check IndexedDB">
    First, the app checks if the audio file already exists in IndexedDB.

    ```typescript
    // From site/src/lib/musicCache.ts:35-46
    async getAudio(trackId: TrackId): Promise<Blob | null> {
        if (!this.db) return null

        return new Promise((resolve, reject) => {
            const transaction = this.db!.transaction(STORE_NAME, "readonly")
            const store = transaction.objectStore(STORE_NAME)
            const request = store.get(trackId)

            request.onerror = () => reject(request.error)
            request.onsuccess = () => resolve(request.result || null)
        })
    }
    ```
  </Step>
  
  <Step title="Fetch if missing">
    If the track isn't cached, it's fetched through the audio proxy endpoint.

    ```typescript
    // From site/src/lib/musicCache.ts:61-78
    async getOrFetch(trackId: TrackId, streamUrl: string): Promise<string> {
        const cached = await this.getAudio(trackId)
        if (cached) {
            return URL.createObjectURL(cached)
        }

        const proxyUrl = `/api/audio/proxy?url=${encodeURIComponent(streamUrl)}`
        const response = await fetch(proxyUrl)
        if (!response.ok) {
            throw new Error(`Failed to fetch audio: ${response.status}`)
        }

        const blob = await response.blob()
        await this.cacheAudio(trackId, blob)

        return URL.createObjectURL(blob)
    }
    ```
  </Step>
  
  <Step title="Store in cache">
    The fetched audio is stored in IndexedDB for future offline access.

    ```typescript
    // From site/src/lib/musicCache.ts:48-59
    async cacheAudio(trackId: TrackId, blob: Blob): Promise<void> {
        if (!this.db) return

        return new Promise((resolve, reject) => {
            const transaction = this.db!.transaction(STORE_NAME, "readwrite")
            const store = transaction.objectStore(STORE_NAME)
            const request = store.put(blob, trackId)

            request.onerror = () => reject(request.error)
            request.onsuccess = () => resolve()
        })
    }
    ```
  </Step>
  
  <Step title="Play immediately">
    The audio plays using a Blob URL, whether from cache or newly fetched.
  </Step>
</Steps>

## Cache initialization

The music cache initializes automatically when the app loads.

```typescript
// From site/src/lib/musicCache.ts:11-33
async init(): Promise<void> {
    if (this.initialized) return
    if (typeof window === "undefined") return // SSR guard

    return new Promise((resolve, reject) => {
        const request = indexedDB.open(DB_NAME, DB_VERSION)

        request.onerror = () => reject(request.error)

        request.onsuccess = () => {
            this.db = request.result
            this.initialized = true
            resolve()
        }

        request.onupgradeneeded = (event) => {
            const db = (event.target as IDBOpenDBRequest).result
            if (!db.objectStoreNames.contains(STORE_NAME)) {
                db.createObjectStore(STORE_NAME)
            }
        }
    })
}
```

## Cache management

You can check cache size and clear the cache if needed.

<CardGroup cols={2}>
  <Card title="Check cache size" icon="database">
    ```typescript
    // From site/src/lib/musicCache.ts:93-114
    async getCacheSize(): Promise<number> {
        if (!this.db) return 0

        return new Promise((resolve, reject) => {
            const transaction = this.db!.transaction(STORE_NAME, "readonly")
            const store = transaction.objectStore(STORE_NAME)
            const request = store.openCursor()
            let totalSize = 0

            request.onerror = () => reject(request.error)
            request.onsuccess = (event) => {
                const cursor = (event.target as IDBRequest).result
                if (cursor) {
                    const blob = cursor.value as Blob
                    totalSize += blob.size
                    cursor.continue()
                } else {
                    resolve(totalSize)
                }
            }
        })
    }
    ```
  </Card>
  
  <Card title="Clear cache" icon="trash">
    ```typescript
    // From site/src/lib/musicCache.ts:80-91
    async clearCache(): Promise<void> {
        if (!this.db) return

        return new Promise((resolve, reject) => {
            const transaction = this.db!.transaction(STORE_NAME, "readwrite")
            const store = transaction.objectStore(STORE_NAME)
            const request = store.clear()

            request.onerror = () => reject(request.error)
            request.onsuccess = () => resolve()
        })
    }
    ```
  </Card>
</CardGroup>

## Service worker installation

The service worker activates automatically and cleans up old caches.

```javascript
// From site/public/sw.js:5-23
self.addEventListener("install", (event) => {
    event.waitUntil(
        caches.open(CACHE_NAME).then((cache) => {
            return cache.addAll(PRECACHE_ASSETS)
        })
    )
    self.skipWait()
})

self.addEventListener("activate", (event) => {
    event.waitUntil(
        caches.keys().then((cacheNames) => {
            return Promise.all(
                cacheNames
                    .filter((name) => name !== CACHE_NAME)
                    .map((name) => caches.delete(name))
            )
        })
    )
    self.clients.claim()
})
```

## Offline playback

When offline, the app falls back to cached content automatically.

```javascript
// From site/public/sw.js:52-66
event.respondWith(
    fetch(event.request)
        .then((response) => {
            if (response.ok) {
                const clone = response.clone()
                caches.open(CACHE_NAME).then((cache) => {
                    cache.put(event.request, clone)
                })
            }
            return response
        })
        .catch(() => {
            return caches.match(event.request)
        })
)
```

<Warning>
YouTube audio streams are not cached due to their dynamic URLs. Only Bandcamp, Spotify previews, and other direct audio sources support offline mode.
</Warning>

## Storage limits

Browsers impose storage quotas on IndexedDB and Cache API. Chrome typically allows:

- Up to 60% of available disk space for origin storage
- Individual audio files are stored as Blobs with no size limit per file
- The entire cache can grow to several GB depending on your disk space

<Tip>
If you're running low on storage, use `musicCache.getCacheSize()` to check usage and `musicCache.clearCache()` to free up space.
</Tip>
